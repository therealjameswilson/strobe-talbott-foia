How it works

The page loads the published manifest (manifest.json with a CSV fallback to manifest.csv) and matches every query token against each record's document ID, date, title, and PDF URL. Search runs entirely in the browser — no server required.

This MVP uses synthetic sample records for workflow testing. It does not publish the full PDF corpus.
